    Deployment Guide¶

    Deploy to production environments.

    Before Deployment¶

    1. Update Version¶

    In AppConstants.cs: 

    public const string AppVersion = "1.05";

    In Models/MobileAppReportConfig.cs: 

    public string Version { get; set; } = "1.05";

    2. Test Thoroughly¶

    # Development test
dotnet watch run
# Test all features in browser

# Distribution test
.\build-distribution.ps1 -Version "1.05"
# Extract and run ReportBuilder.bat
# Test all features

    3. Update Documentation¶

    • Update version in docs
    • Add release notes
    • Update screenshots if needed

    4. Build Distribution¶

    .\build-distribution.ps1 -Version "1.05"

    Creates: ReportBuilder-v1.05.zip

    Deployment Methods¶

    Method 1: Direct Download Link¶

    For internal teams or email distribution:

    1. Upload ZIP to shared location:
   - OneDrive
   - Google Drive
   - Dropbox
   - Sharepoint

2. Share download link

3. Users:
   - Download ZIP
   - Extract
   - Run ReportBuilder.bat

    Method 2: GitHub Release¶

    For open source or team projects:

    1. Create GitHub release:
   - Go to Releases page
   - Click "Create Release"
   - Tag: v1.05
   - Title: Report Builder v1.05
   - Upload: ReportBuilder-v1.05.zip

2. Add release notes:
   - New features
   - Bug fixes
   - Breaking changes

3. Publish release

4. Users download from GitHub

    Method 3: Web Server Hosting¶

    For publicly available download:

    1. Upload ZIP to web server:
   - Your website
   - CDN
   - Azure Storage
   - AWS S3

2. Create download page:
   HTML form with download button

3. Track downloads:
   - Analytics
   - Download counts
   - User feedback

4. Users access from webpage

    Method 4: Windows Update (Advanced)¶

    For enterprise deployment:

    Requires Windows Server Update Services (WSUS) or Group Policy.

    Steps: 1. Create MSI installer (not covered here) 2. Upload to WSUS 3. Deploy via Group Policy 4. Users get auto-update notification

    Method 5: Docker Container¶

    For cloud/server deployment:

    Create Dockerfile:

    FROM mcr.microsoft.com/dotnet/aspnet:9.0
WORKDIR /app
COPY dist/app .
EXPOSE 5000
ENTRYPOINT ["dotnet", "ReportBuilder.Web.dll"]

    Build and push:

    docker build -t reportbuilder:1.05 .
docker run -p 5000:5000 reportbuilder:1.05

    User Instructions¶

    For Windows Users¶

    INSTALLATION INSTRUCTIONS

1. Download ReportBuilder-v1.05.zip

2. Extract:
   - Right-click ZIP
   - Extract All
   - Choose folder

3. Run:
   - Double-click ReportBuilder.bat
   - Wait 10 seconds
   - Browser opens automatically

4. Start using:
   - Fill form
   - Generate PDF

TROUBLESHOOTING:
- Port in use? Edit appsettings.json
- Won't connect? Wait 15 seconds and refresh
- Still stuck? See documentation

    For macOS/Linux Users¶

    INSTALLATION INSTRUCTIONS

1. Download ReportBuilder-v1.05.zip

2. Extract:
   unzip ReportBuilder-v1.05.zip

3. Navigate:
   cd ReportBuilder/app

4. Run executable:
   ./ReportBuilder.Web

   Or with dotnet:
   dotnet ReportBuilder.Web.dll

5. Open browser:
   http://localhost:5000

6. Start using application

    1. Verify Installation¶

    • [ ] Users can extract ZIP
    • [ ] ReportBuilder.bat launches app
    • [ ] Browser opens to correct URL
    • [ ] Can create a test report
    • [ ] Can import a PDF
    • [ ] Can export JSON config
    • [ ] PDF download works

    2. Gather Feedback¶

    • Ask users about experience
    • Collect bug reports
    • Document feature requests
    • Monitor usage patterns

    3. Support Users¶

    • Provide contact info
    • Create FAQ page
    • Answer questions promptly
    • Document common issues

    4. Plan Updates¶

    • Track issues
    • Plan fixes
    • Schedule next release
    • Communicate changes

    Rollback Plan¶

    If deployed version has critical issues:

    Step 1: Communicate¶

    Email to users:
"We've found an issue with v1.05. 
Rolling back to v1.04.
Download updated link below."

    Step 2: Revert¶

    # Go back to previous version
git checkout v1.04-tag

# Build previous version
.\build-distribution.ps1 -Version "1.04"

    Step 3: Redistribute¶

    • Update download link
    • Send notification email
    • Mark v1.05 as problematic
    • Document issue for next release

    Step 4: Fix and Re-release¶

    • Fix the issue
    • Test thoroughly
    • Rebuild: v1.05 patch
    • Release as v1.05.1 or v1.06

    Version Management¶

    Semantic Versioning¶

    Format: MAJOR.MINOR.PATCH

    • 1.0.0 - Initial release
    • 1.0.1 - Patch (bug fix)
    • 1.1.0 - Minor (new feature)
    • 2.0.0 - Major (breaking change)

    Release Checklist¶

    • [ ] Code reviewed
    • [ ] Tests passing
    • [ ] Version updated (3 places)
    • [ ] Documentation updated
    • [ ] Build succeeds: dotnet build
    • [ ] Distribution builds: .\build-distribution.ps1
    • [ ] ZIP tested (extract and run)
    • [ ] Release notes written
    • [ ] Users notified
